# 诊断报告模板

```markdown
# 前端诊断报告

> 生成时间: YYYY-MM-DD HH:mm
> 问题类型: build / runtime / ui / api
> 影响范围: 页面、功能、用户群或环境

## 问题描述

- 用户报告或观察到的现象
- 首次出现时间、环境、分支或版本

## 复现路径

1. 操作步骤或触发命令
2. 期望结果
3. 实际结果

## 收集的证据

- 日志 / stderr / console stack
- 截图、DOM、样式、网络请求或 trace
- 相关配置、依赖版本、环境变量或状态快照

## 假设与验证

| 假设 | 验证方法 | 结果 |
| --- | --- | --- |
| 因为 X，导致 Y | 通过 Z 验证 | 证实 / 证伪 / 待定 |

## 根因

- 最终确认的根本原因
- 为什么已有证据能支持该结论

## 修复内容

- 修改的文件和行为
- 为什么这是最小有效修复

## 验证结果

- 执行的命令、手工路径或浏览器验证
- 修复后观察到的结果

## 剩余风险

- 未覆盖的边界
- 可能回归的区域

## 下一步

- 需要用户决策、后续任务或监控项
```

诊断完成后，将报告保存到 `reports/debug-YYYY-MM-DD-HHmmss.md`，并告知用户报告路径。
